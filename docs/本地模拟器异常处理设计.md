# 本地模拟器 — 异常处理与急救设计

> **日期**: 2026-06-06
> **原则**: 浏览器 ↔ 服务器 ↔ 本地进程 三方交互，任何一环都可能挂。宁可多备份，不留烂摊子。

---

## 2. 异常场景清单

### 2.1 启动前

| # | 场景 | 检测方式 | 处理 |
|---|------|---------|------|
| A1 | Azahar 未配置 | `settings` 表无 `azahar.exe_path` | 前端提示「请先在设置页配置 Azahar 路径」 |
| A2 | Azahar exe 不存在 | `File.Exists()` 返回 false | 返回 400 + 具体路径，前端提示文件不存在 |
| A3 | exe 无执行权限 | Process.Start 抛 Win32Exception | catch → 返回 "无执行权限，请运行 chmod +x" |
| A4 | CIA 未安装 | `content/` 目录不存在或为空 | 返回 400 + 游戏名 + 安装提示 |
| A5 | content 目录存在但 .app 损坏 | FindAzaharContentFile 返回 null | 返回 "CIA 已安装但内容文件缺失，建议重新安装" |
| A6 | 存档文件不存在 | `save.sav` 在 pkmanager 中缺失 | 返回 404 "请先上传存档" |
| A7 | 磁盘空间不足 | 备份时 IOException | 返回 500 + 剩余空间信息，不执行任何写操作 |
| A8 | Azahar 数据目录无写权限 | UnauthorizedAccessException | 返回 "无写入权限: {path}" |
| A9 | 同一存档已有 Azahar 在运行 | pid 文件存在且进程存活 | 返回 "该存档对应的游戏正在运行中 (PID:{pid})" |

### 2.2 启动中

| # | 场景 | 检测方式 | 处理 |
|---|------|---------|------|
| B1 | Process.Start 返回 null | process == null | 返回 "模拟器启动失败（系统拒绝或路径无效）" |
| B2 | Azahar 启动后 3 秒内退出 | 轮询发现 pid 已消失 | 前端显示 "Azahar 启动后立即退出，可能是 ROM/CIA 损坏" + 自动恢复备份 |
| B3 | 启动成功但游戏窗口未出现 | 用户反馈 / 前端超时 | 提供「终止进程」按钮 + 手动恢复 |

### 2.3 运行中

| # | 场景 | 检测方式 | 处理 |
|---|------|---------|------|
| C1 | 用户正常关闭 Azahar | 轮询 pid 消失 + HasExited=true | 正常流程：同步存档 → 恢复备份 |
| C2 | 用户 kill 进程 | 轮询 pid 消失 + HasExited=true | 同正常关闭：尝试同步（可能存档未保存完整） |
| C3 | 进程僵死（zombie） | HasExited=false 但 CPU 0% 持续 >60s | 前端显示警告 + 「强制终止」按钮 |
| C4 | 用户关闭浏览器标签 | beforeunload 事件 | 提醒 "Azahar 仍在运行，关闭页面不会终止模拟器。存档将在模拟器关闭后同步" |
| C5 | 服务器重启 | 内存中 PID 丢失 | 下次查询 local-status 返回 running=false，手动同步 |
| C6 | 前端网络断开 | 轮询失败 | 显示 "与服务器断开，模拟器仍在本地运行。恢复连接后将自动同步" |

### 2.4 关闭后

| # | 场景 | 检测方式 | 处理 |
|---|------|---------|------|
| D1 | 存档同步成功 | HTTP 200 | 恢复备份 → 显示 "存档已同步 ✅" |
| D2 | main 文件不存在（游戏内未存档） | File.Exists(main) = false | 跳过同步，恢复备份，提示 "游戏内未保存，服务器存档未更新" |
| D3 | main 文件损坏 | PKHeX 解析失败 | 保留损坏的 main 副本，恢复备份，提示 "存档解析失败，已保留原始文件" |
| D4 | 上传失败（网络错误） | HTTP 5xx / timeout | 保留 Azahar main 不恢复备份，提示 "同步失败，存档保留在本地。请稍后手动同步" |
| D5 | 备份恢复失败 | File.Copy 异常 | 记录日志，提示 "备份恢复失败，Azahar 中保留的是本次游玩存档" |
| D6 | main 文件被外部修改 | 修改时间与启动时间不一致 | 询问用户 "检测到存档可能被外部工具修改，是否仍同步？" |

### 2.5 并发冲突

| # | 场景 | 检测方式 | 处理 |
|---|------|---------|------|
| E1 | 同一存档重复点击启动 | 内存中已有 (saveFileId, pid) 记录且进程存活 | 返回 409 "游戏已在运行中 (PID:{pid})" |
| E2 | 不同存档同时启动（相同模拟器） | 第二个 Process.Start 前检测 | Azahar 可能支持多实例？默认拒绝，提示「已有模拟器在运行」 |
| E3 | 启动时备份文件被占用 | IOException sharing violation | 重试 3 次（间隔 200ms），失败则报错 |

## 3. 备份与恢复策略

### 3.1 备份目录结构

```
{azahar.data_dir}/pkmanager_backup/
  ├── pid.lock              ← 当前运行的 PID（防止并发）
  ├── 00055D00/             ← 按 title ID 隔离
  │   ├── main.bak          ← Azahar 原始存档备份
  │   └── injected_at.txt   ← 注入时间戳
  └── ...
```

### 3.2 备份操作（启动前）

```
1. 创建 pkmanager_backup/{titleId}/ 目录
2. 如果 Azahar main 文件存在:
     → 复制到 main.bak（覆盖旧备份）
     → 记录时间戳到 injected_at.txt
  如果 main 不存在（Azahar 首次运行该游戏）:
     → 跳过备份，标记 is_first_launch=true
3. 写入 pid.lock = {pid, saveFileId, timestamp}
4. 将 pkmanager save.sav 复制到 Azahar main 路径
```

### 3.3 恢复操作（关闭后）

```
1. 如果 is_first_launch:
     → 删除 Azahar main（pkmanager 注入的，Azahar 不会用）
     → 跳过恢复
  否则:
     → 将 main.bak 复制回 Azahar main 路径（覆盖游玩后的存档）
2. 删除 pid.lock
3. 保留 main.bak 7 天（急救回滚用）
```

### 3.4 急救按钮

前端提供 **「急救恢复」** 按钮，在同步失败或用户需要回滚时：

```
GET /api/Emulator/emergency-restore/{saveFileId}
  → 将 pkmanager_backup/{titleId}/main.bak 恢复为 Azahar main
  → 返回操作结果
```

## 4. 前端轮询设计

```
启动成功后 → 每 2 秒 GET /api/Emulator/local-status/{saveFileId}

返回:
  { running: true, pid: 12345 }      → 显示绿色脉冲点 "Azahar 运行中"
  { running: false }                  → Azahar 已退出 → 自动触发同步
  { running: true, stuck: true }     → 疑似僵死 → 显示黄色警告
  { error: "连接失败" }               → 网络断开 → 显示提示，继续重试

超时逻辑:
  - 超过 30 分钟未关闭 → 不再自动轮询，显示「仍在运行」
  - 超过 60 分钟 → 显示警告「模拟器运行超过 1 小时」
```

## 5. 新游戏流程

```
用户点击「新游戏」
  │
  ├─ 如果 Azahar 已有该游戏 main:
  │     → 备份 main → main.bak
  │
  ├─ 不注入任何存档（Azahar 使用空白/初始存档）
  │
  ├─ Process.Start(azahar, content.app)
  │
  ├─ 轮询等待关闭
  │
  └─ 关闭后:
       ├─ 读取 Azahar main
       ├─ 上传到 pkmanager（CreateNewGame + SyncSave）
       ├─ 创建新存档记录
       └─ 如果有备份 → 恢复 main.bak（保持 Azahar 干净）
```

## 6. 安全底线

| 原则 | 说明 |
|------|------|
| **备份优先** | 任何写操作前先备份，备份失败则中止 |
| **永不删除** | main.bak 保留 7 天，用户可手动清理 |
| **可恢复** | 任何状态下都可执行「急救恢复」 |
| **不阻塞** | Process.Start 不等待，轮询异步 |
| **透明** | 前端每一步都显示状态，出错显示具体原因 |
| **超时兜底** | 每个操作有超时（复制 5s、启动 10s、同步 10s） |
